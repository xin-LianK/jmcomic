# JM Visual

Flutter client plus a small Python API adapter for `jmcomic`.

The app targets iOS, Android, and Web. The current machine does not have Flutter installed, so native platform folders are intentionally generated by `mobile_app/tool/bootstrap_flutter.ps1` after Flutter SDK is available.

## Features

- Album list with category filters, search, and day/week/month rankings.
- Album detail with metadata and chapter list.
- Chapter reader through a backend image proxy/cache.
- Album and chapter download actions with a download queue view.
- Responsive navigation for phones, tablets, and Web.

## Run Backend

The backend installs `jmcomic` from PyPI by default. `JMCOMIC_SOURCE` is only for local development when you want this visual app to run against a checked-out fork of `JMComic-Crawler-Python`.

```powershell
cd D:\demo\workspace\jmcomic_visual\backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
uvicorn jm_server.app:app --host 127.0.0.1 --port 8766 --reload
```

Optional:

```powershell
$env:JMCOMIC_SOURCE="D:\demo\workspace\JMComic-Crawler-Python\src"
$env:JM_OPTION_PATH="D:\path\to\option.yml"
$env:JM_VISUAL_DOWNLOAD_DIR="D:\jm-downloads"
```

## Run Flutter

```powershell
cd D:\demo\workspace\jmcomic_visual\mobile_app
.\tool\bootstrap_flutter.ps1
flutter run -d chrome --dart-define=JM_API_BASE=http://127.0.0.1:8766
```

Android emulator uses a different loopback address:

```powershell
flutter run -d android --dart-define=JM_API_BASE=http://10.0.2.2:8766
```

For a physical phone, use your computer LAN IP, for example `http://192.168.1.20:8766`, and run the backend with `--host 0.0.0.0`.

For an online backend, build the app with your default API address:

```powershell
flutter build apk --release --dart-define=JM_API_BASE=https://api.example.com
```

The API address can also be changed later in the app: `设置 -> API 地址`. Leave it empty to use the build-time/default address.

## Docker

Docker deployment does not need the local `JMComic-Crawler-Python` checkout. The image installs `jmcomic` from PyPI and serves the already-built Flutter Web files plus API on one port.

Build the Web files first. With no `JM_API_BASE`, the browser calls the API on the same origin as the Web page.

```powershell
cd D:\demo\workspace\jmcomic_visual
cd mobile_app
C:\flutter\bin\flutter.bat build web --release
cd ..
docker compose up -d --build
```

Open:

```text
http://127.0.0.1:8088/
```

Persistent folders on the host:

```text
.\downloads  downloaded albums/photos
.\.cache     cover/image cache
```

To use a custom jmcomic option file, edit `docker-compose.yml`, uncomment `JM_OPTION_PATH` and the `option.yml` volume, then restart:

```powershell
docker compose up -d --build
```

Use only content and accounts you are allowed to access. Do not run untrusted `jmcomic` option files.
